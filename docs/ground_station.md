# 地上局ソフトウェア (Ground Station) 解説

## 概要

PC 上で動作する Python スクリプトで、ローバーの**遠隔操縦**と**受信データの可視化**を行います。キーボード入力をリアルタイムに UDP コマンドへ変換してESP32 に送信し、同時にテレメトリやカメラ画像を受信・表示します。

## 実行環境

### 依存パッケージ

| パッケージ | 用途 |
|-----------|------|
| `numpy` | 画像データの配列操作 |
| `opencv-python` (`cv2`) | 画像の表示・リサイズ・回転 |
| `keyboard` | キーボード入力のリアルタイム検出 |

### インストール

```bash
pip install numpy opencv-python keyboard
```

### 実行手順

1. ESP32 の WiFi アクセスポイント `ESP32S3Sense`（パスワード: `Password`）に PC を接続
2. PC の IP アドレスが `192.168.1.2` であることを確認（ESP32 の DHCP で自動割り当て）
3. スクリプトを実行:

```bash
python ground_station/GS.py
```

> **注意**: `keyboard` ライブラリは OS レベルのキー入力を監視するため、Linux 環境では `sudo` が必要な場合があります。

## ソースコード解説

### `GS.py` — 地上局メインスクリプト

#### ネットワーク設定

```python
LOCAL_UDP_IP = '192.168.1.2'     # 地上局のIPアドレス
SHARED_UDP_PORT = 50000           # 地上局の受信ポート
ESP32_UDP_IP = "192.168.1.1"     # ESP32のIPアドレス
ESP32_UDP_PORT = 55555            # ESP32の受信ポート
```

UDP ソケットを `192.168.1.2:50000` にバインドし、受信タイムアウトを 0.5 秒に設定しています。

#### メインループ (`main()`)

```
main() {
  ① receive_telemetry() で ESP32 からのパケットを受信（タイムアウト0.5秒）
  ② keyboard でキー入力を検出し、対応するコマンドを send_command() で送信
  ③ 受信パケットの種類に応じて処理を分岐:
     - packet_number == 0x5C → テレメトリデータ → telemetry_reader()
     - packet_number == 0xFF → フレーム完了 → camera_show() で画像表示
     - それ以外 → 画像データ → image_decode() でバッファに蓄積
}
```

受信とキー入力検出を交互に行うイベントループです。受信タイムアウトにより、パケットが来ない間もキー入力を処理できます。

### 関数の詳細

#### `send_command(command)` — コマンド送信

```python
def send_command(command):
    sock.sendto(command.encode(), (ESP32_UDP_IP, ESP32_UDP_PORT))
```

文字列コマンドを UTF-8 エンコードして ESP32 に UDP 送信します。

#### `receive_telemetry()` — パケット受信

ESP32 から UDP パケットを受信し、以下の3要素に分解します。

```
受信パケット構成:
  [パケット番号(1byte)] [データ(可変長)] [チェックサム(1byte)]
```

- **パケット番号**: パケットの種類を識別（画像チャンネル番号、テレメトリ、フレーム完了合図）
- **データ**: 画像データまたはテレメトリデータ
- **チェックサム**: データ部の合計（8bit）。現在は受信側での検証は未実装

タイムアウト時は `(None, None, None)` を返します。

#### `telemetry_reader(data)` — テレメトリ解析

```python
pressure, temperature, humidity = struct.unpack('<fff', data[1:13])
```

受信データから BME280 のセンサー値を解析します。

- `data[0]` はヘッダ `0x94`（`0x5C` はパケット番号として分離済み）
- `data[1:13]` の 12 バイトを 3 つの float（little-endian）として解釈
- 気圧 (hPa)、温度 (℃)、湿度 (%) をコンソールに表示

#### `image_decode(packet_number, data, image)` — 画像デコード

ESP32 から受信した画像パケットを 240x240 の RGB 画像バッファに書き込みます。

**パケット番号からの位置計算:**

```python
index = (packet_number - 1) // 3 * 12   # 画像上の行位置（12行単位）
rgb = 2 - (packet_number - 1) % 3       # チャンネル: 0=B, 1=G, 2=R
```

- パケット番号 1,2,3 → 行 0〜11 の R,G,B
- パケット番号 4,5,6 → 行 12〜23 の R,G,B
- ... 以降同様に60パケットで全画像をカバー

**データの展開:**

受信データの各バイトは2ピクセル分の4bit値が詰められているため、上位4bitと下位4bitに分解し、16倍して8bit値に拡張します。

```python
# data の各バイトを上位/下位 4bit に分割し、12行x240列に reshape
image[index:index+12, :, rgb] = np.reshape(
    np.ravel(np.array([data // 16, data % 16]).T),
    (12, 240)
) * 16
```

#### カメラ表示関数

3つの表示モードが実装されています（コード内でコメントアウトを切り替えて使用）。

| 関数 | 表示内容 |
|------|---------|
| `camera_show()` | 通常のカラー画像（500x500にリサイズ） |
| `camera_rgb_show()` | 4分割表示: 通常画像 / R単色 / G単色 / B単色 |
| `camera_rgb_diff_show()` | 4分割表示: 通常画像 / 赤み強調 / 緑み強調 / 青み強調 |

`camera_rgb_diff_show()` は各チャンネルから他2チャンネルの平均を引いた「色差分」を表示します。特定の色の物体を探索する際に有用です。

```python
# 赤み = R - (G + B) / 2  （他の色に比べて赤がどれだけ強いか）
redness = np.clip(r16 - ((g16 + b16) // 2), 0, 255)
```

また、`reverese_flag = True` の設定により、カメラ画像は180度回転して表示されます。カメラの物理的な取り付け向きに応じて切り替えてください。

### キー操作一覧

| キー | コマンド | 動作 |
|------|---------|------|
| `W` | `"W"` | 前進 |
| `S` | `"S"` | 後退 |
| `A` | `"A"` | 左旋回 |
| `D` | `"D"` | 右旋回 |
| `G` | `"G"` | 緑LED点滅 |
| `R` | `"R"` | 赤LED点滅 |
| `T` | `"T"` | テレメトリ要求 |
| `B` | `"B"` | 停止 |
| `Q` | — | プログラム終了 |

各キー入力後に 0.1 秒の待機があり、連続送信を抑制しています。
